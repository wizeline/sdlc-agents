# 0. Pre-boarding (Before Day 1)
## 0.1 Account Setup Checklist
- [ ] Email/account provisioned
- [ ] GitHub/GitLab access granted
- [ ] Slack/Teams channels joined
- [ ] VPN/certificate access requested
- [ ] Calendar invites sent

## 0.2 Environment Preparation
- Laptop shipped with standard image
- Required software list provided
- VPN credentials sent securely

---

# 1. Week 1: Foundation
## Day 1: Welcome & Setup
### Morning (HR/IT)
- [ ] Building/remote access orientation
- [ ] Laptop setup and verification
- [ ] Password manager setup
- [ ] 2FA configuration for all services

### Afternoon (Team)
- [ ] Meet the team (coffee chats schedule)
- [ ] Team structure and roles overview
- [ ] First codebase walkthrough (read-only)
- [ ] Development environment setup script

**Day 1 Goal**: Run `hello-world` locally

## Day 2: Tools & Processes
- [ ] Git workflow training (fork/branch/PR)
- [ ] CI/CD pipeline overview
- [ ] Code review guidelines
- [ ] Testing requirements and frameworks
- [ ] Documentation standards

**Deliverable**: Submit first documentation PR (fix a typo)

## Day 3-4: Code Deep Dive
- [ ] Architecture overview (C4 diagrams)
- [ ] Key codebase modules tour
- [ ] Debugging and logging tools
- [ ] Local development data setup

**Deliverable**: Fix a "good first issue" bug

## Day 5: First Week Review
- [ ] Manager 1:1 check-in
- [ ] Mentor feedback session
- [ ] Update onboarding docs with improvements found

---

# 2. Week 2-4: Integration
## Week 2 Goals
- [ ] Complete first feature (small, well-defined)
- [ ] Participate in code review (as reviewer)
- [ ] On-call shadowing (if applicable)
- [ ] Write/run first integration test

## Week 3 Goals
- [ ] Lead a small feature implementation
- [ ] Contribute to technical documentation
- [ ] Cross-team collaboration intro
- [ ] Performance review of own code

## Week 4 Goals
- [ ] Deploy to production (with supervision)
- [ ] Incident response drill (simulated)
- [ ] Present learnings to team (5-min share)
- [ ] 30-day feedback session with manager

---

# 3. Month 2-3: Ownership
## Month 2
- Own a feature from design to deployment
- Mentor next new hire (if applicable)
- Contribute to architectural discussions

## Month 3
- Technical debt reduction project
- Cross-functional project participation
- OKR alignment and goal setting
